Gears Release Notes

Build 0.1.41.0 (2007/05/14)

API Changes

  • scour_utils.js has been removed. Gears now has a "canned" version of this file that you can request with: https://www.google.com/gears?file=common.js. Note: if Gears is not installed this URL is a 404.
  • Factory: create() module names are now 'beta.database', 'beta.localserver', and 'beta.workerpool'. (Formerly they were 'google.com.beta.*'.)
  • LocalServer: manifest entries can match arbitrary query strings using the flag "ignoreQuery": true.
  • LocalServer: store names have the same disallowed characters are Database names.

Build 0.1.38.0 (2007/04/20)

API Changes

  • General: the global Factory object is exposed as (formerly 'gearsFactory') for consistency with other Google AJAX APIs.

New Features

  • Users have control over whether a site can use Gears APIs.
  • A stricter security model is now used for all Gears components (same-origin: identical scheme + host + port).

Resolved Issues

  • Too many to list here.

Known Issues

LocalServer

  • Stale images may be served when the content changes for a captured URL.

WorkerPool

  • The onmessage property is write-only.
  • The onmessage property must be set before sending a message (rather than receiving a message).

Installation

  • Installation on Windows requires administrator rights.
  • The installer is missing user-friendly text.
  • Installation requires a browser restart, but users are not told this.

Other

  • Exceptions in Firefox are not instanceof nsIConsoleError. This makes them not work properly with Firebug and JS console. As a workaround, you can catch the errors and print their .message property, or override all Objects' toString() to check if there is a message property and if so, print it.
